# 법적 근거 매핑

각 PII 카테고리가 한국 법령 어느 조항에 근거하는지 정리. 검출 결과의
`DetectionResult.legal_basis` 와 `legal/mapping.py` 의 단일 진실 소스를
일관성 있게 유지하기 위함.

## 카테고리별 매핑

| 라벨 | 카테고리 | 법적 근거 |
|------|----------|-----------|
| `RRN` | 고유식별정보 | 개인정보보호법 제24조의2 (주민등록번호 처리 제한) |
| `FRN` | 고유식별정보 | 개인정보보호법 시행령 제19조; 출입국관리법 제31조 |
| `PASSPORT` | 고유식별정보 | 개인정보보호법 시행령 제19조 |

## 공통 법령

- **개인정보보호법**
  - 제2조 (정의): 개인정보·가명정보 일반 정의
  - 제15조: 개인정보의 수집·이용
  - 제23조: 민감정보 처리 제한
  - 제24조 / 제24조의2: 고유식별정보 / 주민등록번호 처리 제한
  - 제28조의2~5: 가명정보 처리 특례 (가역 가명화 근거)
  - 제29조: 안전조치의무 (처리 이력 기록 요건 — 본 라이브러리의 증명서 기능)
- **개인정보보호법 시행령**
  - 제18조: 민감정보 범위
  - 제19조: 고유식별정보 범위 (주민등록번호·여권번호·운전면허번호·외국인등록번호)
- **개인정보보호위원회 고시·가이드라인**
  - 「가명정보 처리 가이드라인」
  - 「개인정보 비식별 조치 가이드라인」 (가명처리 / 일반화 / k-익명성 / l-다양성 등)

## 처리 모드와 법령 매핑

| ProcessingMode | 차단 기준 | 적용 시나리오 |
|----------------|-----------|---------------|
| PARANOID | 위험도 LOW 이상 / 신뢰도 0.5 이상 | 외부 제3자 공개·자료 첨부 시 |
| STRICT | MEDIUM / 0.7 | 표준 운영 (제29조 안전조치) |
| BALANCED | HIGH / 0.8 | 내부 분석·연구 |
| PERMISSIVE | CRITICAL / 0.95 | 권한 있는 사용자의 검토 단계 |
| AUDIT | 차단 안 함 | 감사·검출 보고만 (제29조 처리 이력) |
