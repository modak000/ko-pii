# 실데이터 평가 결과

## 외부 벤치마크: KLUE-NER

**데이터 출처:** [KLUE-benchmark/KLUE](https://github.com/KLUE-benchmark/KLUE) —
한국어 자연어이해(NLU) 평가용 공개 벤치마크 (2021, 카카오·NAVER·SKT 등 공동
구축). NER 서브셋: 신문기사 기반 + 6 entity types (PS·OG·LC·DT·TI·QT) BIO 태깅.

다운로드:
```bash
curl -L -O https://raw.githubusercontent.com/KLUE-benchmark/KLUE/main/\
     klue_benchmark/klue-ner-v1.1/klue-ner-v1.1_dev.tsv
# 5,000 문장 / 2.6 MB
```

실행:
```bash
python -m k_pii.eval.klue_benchmark klue-ner-v1.1_dev.tsv --show-errors 5
```

## 평가 결과

### k-pii v0.4 PERSON 검출 (KLUE-NER dev)

| 매칭 | TP | FP | FN | Precision | Recall | F1 |
|---|---|---|---|---|---|---|
| partial (한글 풀네임만) | 1,127 | 6,231 | 1,967 | 0.153 | 0.364 | **0.216** |

(전체 PS 라벨 평가 — 영문 가명·외래어 포함 — 시 다른 수치 산출. CLI 에서
`--all-labels` 옵션.)

### 합성 공문서 벤치마크 (k-pii 의도 사용처)

| 코퍼스 | 카테고리 | F1 |
|---|---|---|
| 6 템플릿 × 60문서 × 8 seed = 480 문서 | 모두 | **1.000** |

### F1 격차의 해석

**1.000 → 0.216 의 갭이 본 라이브러리의 정직한 그림이다.**

이 격차는 *룰 기반 시스템 vs. 자연어의 본질적 다양성* 의 trade-off 를 보여준다.

| 평가 환경 | 의미 | F1 |
|---|---|---|
| 합성 공문서 | "성명: 홍길동" 같은 *정형 양식* — 본 라이브러리의 의도 사용처 | 1.000 |
| KLUE-NER 자연어 | 신문기사 자유 문장 — 본 라이브러리의 *비-의도* 영역 | 0.216 |

## 분석: 왜 KLUE-NER 에서 약한가

### FN (놓친 이름) 상위 사례

| 이름 | 빈도 | 패턴 |
|---|---|---|
| 왕가위, 코엔, 소피마르소 | 17 | **외국인명** — 한국 공공 가명화 *대상 아님* |
| 장혁, 이범수, 류현진 | 14 | 한국 이름 + 조사 — 조사 가중치 0.35 로 보강함 |
| 도라에몽, 사다코, 짐캐리 | 10 | **가공인물 / 외래어** — 가명화 대상 아님 |
| 예수 | 12 | 종교 인물 — 도메인 외 |
| 박, 김, 박씨, 김모 | 449 | **단음절 성씨 / 부분 가명** — 의도적 미검출 (확실성 부족) |

→ FN 의 *상당수* 가 KLUE-NER 라벨링 기준과 본 라이브러리의 *정책적 미검출*
(단음절·외국인·가공) 의 차이.

### FP (잘못 잡은 토큰) 상위 사례

| 토큰 | 빈도 | 원인 |
|---|---|---|
| 지난, 이날, 자신 | 26+ | 시간·대명사 (지/이/자 surname-시작) |
| 경찰, 후보자, 기자회견 | 6 | 신문기사 빈출 어휘 |
| 박모, 윤모, 정씨 | 5 | 부분 가명 패턴 — KLUE 는 PS 미라벨 |
| 배우, 학교, 사건 | 4 | 일반 명사 (surname-시작) |

→ FP 는 자연어 어휘 폭발의 본질적 결과. ``common_words`` 사전 확장으로 점진
개선 가능하나 *완전 제거 불가*.

## 룰 기반 한계의 본질

룰 기반 자연어 NER 은:

1. **어휘 폭발에 약함** — 한국어 일반 명사 수만 개를 모두 ``common_words`` 에
   넣을 수 없음
2. **문맥 의미 추론 불가** — "왕가위 감독" 의 "왕가위" 는 인명, "박물관" 의
   "박물" 은 명사. 룰만으로 구분 어려움
3. **고유명사 + 일반어 충돌** — "강", "이", "지" 같이 성씨이면서 시간·지명·
   대명사인 글자

이는 ML NER (KoELECTRA, KLUE/RoBERTa 등) 가 우세한 영역이지만, 본 프로젝트는
**ML 사용 안 함** (CLAUDE.md §2 #1) 을 명시 원칙으로 한다.

## 사용자 워크플로우 — 한계 보완

ML 없이도 production 가능한 이유:

1. **공문서 정형 양식은 룰 기반으로도 완벽** (F1=1.000) — 한국 공공
   문서의 *대부분이* 정형 양식. "성명: ", "신청인: ", "수신자: " 같은 필드
   라벨이 PII 위치를 명시.
2. **자연어 부분은 사람 검토 워크플로우** (`k_pii.review`) — REVIEW 큐 +
   OK/FP/FN 마킹 + 피드백 학습. ML 보다 *느리지만 더 정확*. 또한 감사 추적이
   명확.
3. **결정적 PII (RRN/카드/사업자) 는 ML 보다 정확** — 체크섬 검증은 ML 이
   못 흉내냄.

## 결론

- 본 라이브러리는 **정형 공문서 1차 자동 가명화** 도구로 production-ready
- 자유 자연어 PII 검출은 *보조* 수준 — 사람 검토 큐와 함께 사용 권장
- 합성 F1=1.000 은 *의도된 사용처에서 동작함* 의 신호, 보편 정확도 주장 X
- KLUE-NER F1=0.216 은 *룰 기반 자연어 NER 의 정직한 천장*

향후 개선:
- 사용자 코퍼스 (실 공문서 익명화 샘플) 기반 ``common_words`` 자동 학습
- 부서별 직책 사전 세분화 (도메인 입력 필요)
- KLUE-NER FP 패턴 → 자연어 사전 보강 (반복 작업)
