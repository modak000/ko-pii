# 커버리지 — 어떤 문서에서 어떤 PII 를 잡는가

## PII 카테고리 (21종)

### 결정적 PII (Phase 1) — 체크섬·포맷 결정적

| 라벨 | 포맷 | 검증 |
|------|------|------|
| RRN | `YYMMDD-Sxxxxxx` (13자리) | 날짜 + 가중합 체크섬 |
| FRN | `YYMMDD-Sxxxxxx` (13자리, S∈5·6·7·8) | 날짜 + 가중합 체크섬 |
| BUSINESS_REG | `XXX-XX-XXXXX` (10자리) | 국세청 가중합 |
| CORP_REG | 13자리, 6-7 사이 하이픈 | Luhn-like |
| DRIVER_LICENSE | `RR-YY-NNNNNN-CC` (12자리) | 지방청 코드(11~28) |
| PASSPORT | `[A-Z]{1,2}\d{8}` | 형식 only (체크섬 미공개) |
| CARD | 13~19자리 | Luhn mod-10 |
| MEDICAL_INSURANCE | 11자리 | "건강보험/의료보험/보험증" 키워드 25자 |

### 비검증 PII (Phase 2) — 휴리스틱·anchor

| 라벨 | 포맷 | anchor |
|------|------|--------|
| PHONE | 010~019/02/031~064/070 + 국제 `+82` | — |
| FAX | 일반 전화 포맷 | "팩스/FAX/전송" 키워드 12자 |
| EMAIL | RFC 5322 실용 부분집합 | — |
| POSTAL_CODE | 5자리 / 6자리 레거시 | 5자리는 "우편번호" 키워드 필수 |
| IP | IPv4 / IPv6 (RFC 4291) | — |
| VEHICLE | `NN(N)[가-힣]NNNN` 신형 | — |
| URL | http(s) | — |
| ADDRESS | 도로명 / 지번 (동·읍·면·리) | 시·도/시·군·구 prefix 또는 "주소" 키워드 |
| ACCOUNT | 10~16자리 + 하이픈 | "계좌" 키워드 |

### 컨텍스트 (Phase 3)

| 라벨 | 신호 |
|------|------|
| PERSON | 성씨 사전 + 필드 라벨 + 직책 인접 + 결정적 PII 인접 + 조사 + 누적 사전 |

### 도메인 특화 (Phase 4)

| 라벨 | 도메인 | 포맷 |
|------|--------|------|
| DOC_ID | 정부 공문서 | `<기관>-<부서>-YYYY-NNNNN` |
| PETITION_ID | 민원 | `YYYY-민원-NNNNN` / `정보공개-YYYY-NNNNN` 등 |
| EMPLOYEE_ID | 인사 | 4~12자리 숫자 + 사번/공무원번호/교번 키워드 |

## 문서 유형별 커버리지

### 정부 공문서 (결재·보고·회의록·협조)

- ✅ RRN, FRN (신청인·민원인)
- ✅ PHONE, EMAIL (연락처·이메일)
- ✅ ADDRESS (도로명 / 지번)
- ✅ PERSON (수신자·참조·기안자·결재자, 직책 인접)
- ✅ DOC_ID (문서번호)
- ⚠️ 직책 사전 — 현재 일반 시드. 부처별 세분화는 사용자 입력 필요.

### 민원 응대 문서

- ✅ RRN, PHONE, EMAIL (민원인)
- ✅ ADDRESS (주소)
- ✅ PERSON (민원인·답변자)
- ✅ PETITION_ID (민원번호·정보공개 청구번호)
- ⚠️ 민원 양식 자체의 구조화된 표는 현재 평문 가정.

### 인사 문서 (이력서·평가서·인사카드)

- ✅ RRN (주민등록번호)
- ✅ PHONE, EMAIL
- ✅ PERSON (성명·평가자)
- ✅ EMPLOYEE_ID (사번·공무원번호·교번)
- ⚠️ 경력·학력 자유 텍스트의 세부 필드 (재직기간/학교명) 는 현재 비대상.

### 의료 문서

- ⚠️ MEDICAL_INSURANCE (건강보험증) 만 지원
- ⏸ KCD 진단코드, EDI 약품코드 사전은 사용자 도메인 입력 대기 (TODO.md)

## 알려진 한계
